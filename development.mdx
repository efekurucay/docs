## Configuration (docs.json)

### Basic Configuration

```json
{
  "name": "Your Documentation",
  "logo": {
    "light": "/logo/light.svg",
    "dark": "/logo/dark.svg",
    "href": "https://yoursite.com"
  },
  "favicon": "/favicon.svg",
  "colors": {
    "primary": "#2AB673",
    "light": "#55D799",
    "dark": "#117866"
  },
  "topbarLinks": [
    {
      "name": "Documentation",
      "url": "https://docs.yoursite.com"
    }
  ],
  "topbarCtaButton": {
    "name": "Get Started",
    "url": "https://dashboard.yoursite.com"
  }
}
```

### Navigation Configuration

```json
{
  "navigation": [
    {
      "group": "Getting Started",
      "pages": ["introduction", "quickstart"]
    },
    {
      "group": "Features",
      "pages": ["features/one", "features/two"]
    }
  ]
}
```

### Global Settings

```json
{
  "global": {
    "anchors": [
      {
        "anchor": "Home",
        "href": "https://yoursite.com",
        "icon": "globe"
      }
    ],
    "api": {
      "baseUrl": "https://api.yoursite.com",
      "auth": {
        "method": "bearer"
      }
    }
  }
}
```

## Page Structure

### Metadata Headers

Every MDX page should start with metadata:

```mdx
---
title: "Your Page Title"
description: "A brief description of the page"
---
```

Additional metadata options:
```mdx
---
sidebarTitle: "Shorter Title" # Optional - different sidebar title
api: "GET /api/endpoint" # For API pages
openapi: "GET /pets" # OpenAPI endpoint
---
```

## Best Practices

### Documentation Structure

1. **Organize Content Hierarchically**
   - Group related content together
   - Use clear, descriptive titles
   - Keep navigation depth reasonable

2. **Writing Style**
   - Use clear, concise language
   - Include practical examples
   - Break down complex topics

3. **Component Usage**
   - Use appropriate components for content type
   - Maintain consistent styling
   - Don't overuse decorative elements

### SEO Optimization

<Note>
  Always include:
  - Descriptive page titles
  - Clear meta descriptions
  - Relevant keywords
  - Proper heading hierarchy
</Note>
